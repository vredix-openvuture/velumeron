## Features

- **Hyprland** вҖ” Wayland compositor (Lua-based config via `hypr.lua/`)
- **Waybar** вҖ” Modular status bar with interactive setup wizard
- **Wallpaper theming** вҖ” `awww` + `wallust` auto-generate color schemes from wallpapers
- **Submaps** вҖ” Window management, navigation, quickstart, and developer keybind layers
- **Floating window cascading** вҖ” Smart cascade + clamp for floating windows
- **Per-monitor bars** вҖ” Each monitor gets its own Waybar config built by `waybar.sh`
- **Bluetooth notifications** вҖ” systemd service sends desktop notifications on connect/disconnect
- **Lock screen** вҖ” `hyprlock` with workspace save/restore and playback pause
- **Session menu** вҖ” `wlogout` for logout/lock/sleep/reboot/shutdown
- **Rofi wallpaper picker** вҖ” Browse wallpapers with thumbnails via Rofi

## Setup

### 1. Hyprland

Run the interactive configurator to set up your device-specific settings:

```bash
bash ~/.config/vutureland/.setup/hyprland.sh
```

This generates `hypr/user_settings.conf` with your monitors, cursor, keybindings, and startup apps.

### 2. Waybar

Run the interactive setup to build a Waybar config for each monitor:

```bash
bash ~/.config/vutureland/.setup/waybar.sh
```

Select monitor вҶ’ position вҶ’ style вҶ’ modules for each slot (left / center / right), then save.
To rebuild after changes:

```bash
bash ~/.config/vutureland/.setup/waybar.sh --rebuild
```

### 3. Wallpaper

Set a wallpaper and auto-generate colors:

```bash
fish ~/.config/vutureland/assets/scripts/wallpaper-set.fish ~/path/to/wallpaper.png
```

Or pick one via Rofi (mapped to a keybind in the quickstart submap).

## Waybar Modules

Each module lives in `waybar-modular/modules/<name>/` with:
- `config.json` вҖ” Waybar module config
- `style.css` вҖ” Module styling
- `module.md` вҖ” Metadata (name, description, alias, slot restriction)

To restrict a module to a specific bar slot, add to its `module.md`:
```
specific = "left"   # or "center" / "right"
```

## Bluetooth Notifications

Enable the bt-notify systemd service for connection/disconnection alerts:

```bash
# Copy the service file (see header of assets/scripts/bt-notify.sh)
systemctl --user daemon-reload
systemctl --user enable --now bt-notify.service
```
