# Wallpaper System

Wallpapers are stored in two directories and paired by a shared ID so horizontal and vertical monitors always show matching images.

## Storage

```
assets/wallpaper/
├── horizontal/   wp_<ID>_hor.<ext>  or  wp_<ID>_vid_hor.<ext>
└── vertical/     wp_<ID>_ver.<ext>
```

IDs are 6-character alphanumeric strings (e.g. `wp_a3f9bc_hor.jpg`).

Supported formats: `.jpg`, `.jpeg`, `.png`, `.webp`, `.mp4`, `.webm`, `.mkv`, `.avi`, `.mov`

Video wallpapers on horizontal monitors use `mpvpaper`. Vertical monitors only support static images.

## Setting a wallpaper

```bash
# Set by file (counterpart auto-detected by ID)
bash ~/.config/vutureland/assets/scripts/wallpaper-set.sh /path/to/wp_abc123_hor.jpg

# Set horizontal and vertical explicitly
bash ~/.config/vutureland/assets/scripts/wallpaper-set.sh \
  --hor /path/to/wp_abc123_hor.jpg \
  --ver /path/to/wp_abc123_ver.jpg

# Set without the preview animation
bash ~/.config/vutureland/assets/scripts/wallpaper-set.sh --no-showcase --hor ...
```

## Random wallpaper

```bash
bash ~/.config/vutureland/assets/scripts/wallpaper-random.sh
```

## GUI

The **Wallpaper** tab in `gui/main.py` shows thumbnails in three views:
- **Sets** — pairs with both horizontal and vertical
- **Horizontal** — all horizontal images
- **Vertical** — all vertical images

Click any thumbnail to apply. Use **Add…** to import new files.

## Thumbnail cache

Thumbnails are stored in `~/.cache/vutureland/wallpaper-thumbs/` and generated by `rofi/assets/generate-thumbnail.sh`.

## Theme names

Optional display names are stored in `assets/wallpaper/theme-names.txt`:
```
wp_abc123  My Landscape
wp_def456  Night City
```
