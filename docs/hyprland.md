# Hyprland Configuration

Vutureland uses a Lua-based Hyprland config split across several files in `hypr.lua/`.

## Key files

| File | Purpose |
|---|---|
| `hypr.lua/hyprland.conf` | Main entry point — sources all other files |
| `hypr.lua/user_settings.lua` | **User-editable settings** (monitors, keys, autostart, window rules) |
| `hypr.lua/colors.lua` | Auto-generated by wallust — do not edit manually |
| `hypr.lua/modules/` | Theme modules: animations, decorations, keybinds, layouts, etc. |
| `hypr.lua/hyprlock.conf` | Hyprlock screen locker config |
| `hypr.lua/hyprlock-themes/` | Alternative hyprlock themes |

## user_settings.lua sections

Delimited by `-- BEGIN <SECTION>` / `-- END <SECTION>` markers, which the GUI uses to find and rewrite sections without touching the rest of the file.

- **MONITORS** — output, mode, scale, transform, position, VRR, bit depth
- **PERIPHERALS** — function key bindings, cursor theme + size
- **AUTOSTART** — background daemons + workspace startup apps
- **WINDOWRULES** — floating regex, opacity regex

## Editing via GUI

Open the **Hyprland** tab in `gui/main.py`. Changes are written to `user_settings.lua` and `hyprctl reload` is called automatically.

## Editing manually

Edit `user_settings.lua` directly, then run:
```bash
hyprctl reload
```

## Hyprlock

Launch via:
```bash
bash ~/.config/vutureland/assets/scripts/launch-hyprlock.sh
```

Switch themes: `hypr.lua/hyprlock-themes/` contains alternative `.conf` files. The active theme is symlinked as `hypr.lua/active-hyprlock.conf`.
