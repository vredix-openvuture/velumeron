# Waybar – Overview

## Quick reference

```
modules/*            ←  add modules here
waybar.sh            →  auto-includes.json is updated
waybar.sh --rebuild  →  rebuild output/ + start Waybar
launch-waybar.sh     →  restart Waybar with current output/
```

---

## Directory structure

```
waybar-modular/
├── assets/
│   ├── waybar.md              this file
│   └── mpris.sh, ...          helper scripts for modules
│
├── base/
│   └── base-top/
│       ├── dock.config.json   base bar config (layer, margin, ...)
│       └── dock.css           base styling (imports colors_gtk.css)
│
├── containers/
│   └── audio-drawer/
│       ├── container.md       metadata (name, description)
│       ├── definition.json    Waybar group definition (drawer, modules)
│       └── style.css          styling for this container
│
├── groups/
│   ├── auto-includes.json     AUTO-GENERATED: all modules/*/config.json
│   ├── bars.json              monitor → group mapping (set once per device)
│   └── groups-dock-top.json   group layout per monitor (left/center/right)
│
├── modules/
│   └── clock/
│       ├── config.json        Waybar module configuration
│       ├── style.css          module styling
│       └── module.md          metadata (name, description)
│
└── output/                    AUTO-GENERATED by waybar.sh --rebuild
    ├── merged-config.json     Waybar starts with this file (-c)
    └── merged-style.css       Waybar starts with this file (-s)
```

---

## Workflow: Adding a new module

1. Create folder: `modules/my-module/`
2. Create files: `config.json`, `style.css`, `module.md`
3. Run `waybar.sh` (or `waybar.sh --rebuild`)
   → The module is automatically added to `auto-includes.json`
4. Add the module to the desired group in `groups/groups-dock-top.json`
5. `waybar.sh --rebuild` → Waybar restarts with the new module

---

## Workflow: Rebuild (waybar.sh --rebuild)

```
1. update_auto_includes()
   → scan modules/*/config.json
   → add missing entries to auto-includes.json

2. build_config()
   → read bars.json (monitor, groups file, left/center/right)
   → load base-top/dock.config.json as base
   → per bar: set include to groups-dock-top.json
   → write output/merged-config.json

3. build_style()
   → import base-top/dock.css
   → import all modules/*/style.css
   → import all containers/*/style.css
   → write output/merged-style.css

4. Kill and restart Waybar with:
   waybar -c output/merged-config.json -s output/merged-style.css
```

---

## Include chain (how Waybar reads the configs)

```
output/merged-config.json           (array of bar configs)
  └─ include: groups-dock-top.json  (group layout)
       └─ include: auto-includes.json  (all modules)
            └─ include: modules/clock/config.json
            └─ include: modules/workspaces/config.json
            └─ ...
```

Waybar processes `include` recursively: each included file
can itself have further `include` entries.

## Manual file editing

| File                         | When to edit                              |
|------------------------------|-------------------------------------------|
| `groups/bars.json`           | New device / change monitor               |
| `groups/groups-dock-top.json`| Change group layout (what goes where)     |
| `modules/*/config.json`      | Adjust module behavior                    |
| `modules/*/style.css`        | Adjust module styling                     |
| `base-top/dock.config.json`  | Bar properties (margin, layer, ...)       |
| `base-top/dock.css`          | Global styling (colors, fonts, ...)       |

**Do not edit manually:**
- `groups/auto-includes.json` (managed by `waybar.sh`)
- `output/merged-config.json` (generated by `waybar.sh --rebuild`)
- `output/merged-style.css` (generated by `waybar.sh --rebuild`)

---

## Scripts

| Script                         | What it does                                    |
|--------------------------------|-------------------------------------------------|
| `waybar.sh`                    | Checks for new modules, updates auto-includes   |
| `waybar.sh --rebuild`          | Full rebuild + restart Waybar                   |
| `waybar.sh --rebuild --debug`  | Rebuild with detailed output                    |
| `launch-waybar.sh`             | Restart Waybar with current output/             |

## bars.json – Format

```json
[
  {
    "output":      "DP-2",
    "groups-file": "groups-dock-top.json",
    "style":       "dock",
    "position":    "top",
    "left":        "group/left_pri_hor",
    "center":      "group/center_pri_hor",
    "right":       "group/right_pri_hor"
  }
]
```

- `output`: monitor name (e.g. DP-2, HDMI-A-1)
- `groups-file`: filename in `groups/` with group definitions
- `style`: style name (corresponds to `base-top/dock.config.json`)
- `position`: top / bottom / left / right
- `left/center/right`: group keys from the groups file
